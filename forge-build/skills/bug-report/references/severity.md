# 심각도 판단 기준

| 등급 | 기준 | 예시 |
|------|------|------|
| **Critical** | 서비스 중단 / 데이터 손실 / 보안 취약점 / 결제 오류 | DB 다운, 개인정보 노출, 결제 실패 |
| **High** | 핵심 기능 불작동 / 대다수 사용자 영향 / 우회 불가 | 로그인 불가, 저장 안 됨, 게임 크래시 |
| **Medium** | 기능 부분 불작동 / 소수 사용자 영향 / 우회 가능 | UI 깨짐, 특정 조건에서만 오류, 성능 저하 |
| **Low** | 미관·UX 문제 / 긴급성 낮음 | 오타, 색상 불일치, 경고 메시지 누락 |

## 판단 트리

```
서비스/데이터/보안/결제 영향? → Critical
핵심 기능 완전 불작동 + 우회 불가? → High
부분 불작동 또는 우회 가능? → Medium
그 외 미관·UX? → Low
```

## 우선순위 처리 기준

| 등급 | 목표 대응 시간 | 처리 방식 |
|------|-------------|---------|
| Critical | 즉시 (1시간 내) | `/forge-fix` hotfix 즉시 진입 |
| High | 당일 | `/qa --mode=hotfix` 또는 다음 sprint P0 |
| Medium | 1주 내 | 다음 sprint backlog |
| Low | 다음 릴리즈 | backlog 낮은 우선순위 |
